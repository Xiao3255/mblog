# 安装nvm和npm

TIP

如果你曾经安装过node，请先将其卸载。nvm官网不建议使用brew安装nvm，安装node时建议安装lts稳定版本。

# 安装nvm

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.35.0/install.sh | bash

提示nvm command not found，curl安装的nvm会自动写入path，如果你使用brew安装，你需要手动将其写入(.zshrc)或者(.bashrc)或者(.bash_profile)

如

vim ~/.zshrc 或 open ~/.zshrc

写入以下代码，退出保存

export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"  # This loads nvm
[ -s "$NVM_DIR/bash_completion" ] && \. "$NVM_DIR/bash_completion"  # This loads nvm bash_completion

让配置生效

source ~/.zshrc #if you config .zshrc

安装完查看是否安装成功

nvm --version

# 安装node

安装lts稳定版node，最新为Dubnium

nvm install --lts='Dubnium'

# 指定版本(具体看终端提示操作)

nvm use XX

# 查看是否安装成功

nvm ls

# nvm常用命令

查看当前版本

nvm current

失活当前版本

nvm deactivate

列出已安装实例

nvm ls

卸载包

nvm uninstall XX

...

# npm常用命令

查看已安装的全局包

npm ls -g --depth=0

...

# 优化

  • 新建终端时,由于载入nvm,导致终端载入超过1秒。在nvm安装时,要求在在 ~/.bash_profile, ~/.zshrc, ~/.profile, ~/.bashrc 中的一个载入以下内容

从官方 issue#860上了解到, 可以使用 --no-use 和 手动指定默认 node 路径来加快执行速度

export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh" --no-use  # This loads nvm
export PATH=$HOME/.nvm/versions/node/v10.16.3/bin/:$PATH
[ -s "$NVM_DIR/bash_completion" ] && \. "$NVM_DIR/bash_completion"  # This loads nvm bash_completion

默认路径可以通过 nvm which default 获取

  • npm换源

临时使用淘宝镜像(已安装express为例)

npm --registry https://registry.npm.taobao.org install express

永久使用淘宝镜像

npm config set registry https://registry.npm.taobao.org

使用官方镜像

npm config set registry https://registry.npmjs.org/

查看npm源地址

npm config get registry
