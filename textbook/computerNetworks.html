# 因特网概述

# 网络的网络

网络是由若干结点和连接这些结点的链路组成。

TIP

网络中的结点可以是计算机、集线器、交换机和路由器等。 链路即传输介质如网线、HOST 主机、终端设备等.

网络的网络：网络还可以通过路由器互连起来，这样就构成了一个覆盖范围更大的网络，即互联网。

因特网是世界上最大的互联网络，习惯上把因特网上的计算机都称为主机（host）。

TIP

路由器是一种特殊的计算机，它是连接不同的网络的专用设备，用户并不直接使用路由器处理的信息。因此不能把路由器称为主机。

# 基本概念

网络把许多计算机连接在一起，而互联网则把许多网络连接在一起。因特网是世界上最大的互联网。

物理网络：直接连接计算机的网络。

逻辑网络：由物理网络集合构成的网络（即互联网）。

TIP

把计算机简单地在物理上连接起来并不能达到计算机之间信息交换的目的，我们还必须在计算机上安装许多使计算机能够交换信息的软件才行。

# 因特网发展的三个阶段

# internet 和 Internet

  • internet(互联网或互连网)是一个通用名词，它泛指由于多个计算机网络互连而组成的网络。

  • Internet（因特网）是一个专有名词，它指当前全球最大的、最开放的、由众多网络相互连接而成的特定的计算机网络，它采用TCP/IP 协议簇作为通信的规则，且其前身为美国的 ARPANET。

# 三级结构的因特网 (stage 2)

主干网、地区网和校园网（或企业网）

# 多层次 ISP 结构的因特网 (stage 3)

ISP即 Internet Service Provider，因特网服务提供者（因特网服务提供商）。

ISP 可以从因特网管理机构申请到成块的 IP 地址，同时拥有通信线路（大的 ISP 自己建设通信线路，小的 ISP 则向电信公司租用通信线路），以及路由器等连网设备。

任何机构和个人只要向 ISP 交纳规定的费用，就可以从 ISP 得到所需的 IP 地址，并通过该 ISP 联入到因特网。我们通常说的上网就是指“通过某个 ISP 接入到因特网”。

TIP

IP 地址的管理机构不会把单个的 IP 地址分配给某个用户（不“零售”IP 地址）,而是把一批 IP 地址有偿分配给经审查合格的 ISP（只“批发”IP 地址）。

# ISP 也分成不同的层次

根据提供服务的覆盖面积以及所拥有的 IP 地址数目的不同，ISP 可分为三个层次。

  • 第一层 ISP（tier-1 ISP）：最高级别，服务面积最大，一般能够覆盖国际性区域范围，并拥有高速链路和交换设备。通常也被称为因特网主干网，并直接与其他第一层 ISP 相连。

  • 第二层 ISP：第二层 ISP 和一些大公司都是第一层 ISP 的用户，通常具有区域性或国家性覆盖规范，少与第一层 ISP 相连接。

  • 第三层 ISP：又称本地 ISP，第二层 ISP 的用户，只拥有本地范围的网络。一般的校园网或企业网，以及住宅用户或无线移动用户等，都是第三层 ISP 的用户。

# 对等（peer）

一个 ISP 也可以选择与其他同层次 ISP 相连，当两个同层次 ISP 彼此直接相连时，它们被称为是对等的。

TIP

一旦某个用户接入到因特网，那么他能够成为一个 ISP。他需要做的就是购买一些如调制解调器或路由器这样的设备，让其他用户能够和他相连接。一个 ISP 可以很方便地在因特网拓扑上增添新的层次和分支。

# 因特网的标准化工作

RFC（Request for comment），所有的因特网标准都是以 RFC 文档的形式在因特网上发表。所有的 RFC 文档都可以从因特网上免费下载。

RFC 按收到的时间先后从小到大编上序号（即 RFC xxxx，这里的 xxxx 是阿拉伯数字）。一个 RFC 文档更新后就使用一个新的编号，并在文档中指出原来老编号的文档已成为陈旧的。

TIP

并非所有的 RFC 文档都是因特网标准，只有一小部分 RFC 文档最后才能变成因特网标准。

# 制定因特网的正式标准的 4 个阶段

  1. 因特网草案（Internet Draft）---在这个阶段还不是 RFC 文档
  2. 建议标准（Proposed Standard）---从这个阶段开始就成为 RFC 文档
  3. 草案标准（Draft Standard）
  4. 因特网标准（Internet Standard）

# 因特网的组成

因特网从功能上可以分为一下两大块

  • 边缘部分：由所有连接在因特网上的主机组成。这部分是直接面向用户的，用来运行各种网络应用，为用户直接提供电子邮件、文件传输、网络音/视频等服务。
  • 核心部分：由大量网络和连接这些网络的路由器组成。这部分是为边缘部分提供服务的（提供连通性和数据交换）。

TIP

在因特网核心部分的 router 之间一般都是用高速链路相连接，而在网络边缘的主机接入到核心部分则通常以相对较低速率的链路相连接。

# 因特网的边缘部分

处在因特网边缘的部分就是连接在因特网上的所有主机（又称为端系统 End System）。

# 主机 A 和主机 B 进行通信

实际上指的是“运行在主机 A 上的某个程序和运行在主机 B 上的另一个程序通信”。由于进程就是运行着的程序，因此这也就是指：“主机 A 上某个进程和主机 B 上的另一个进程通信”。这种比较严密的说法通常可以简称为“计算机之间通信”。

# Client and Server

在计算机网络上下文中“C/S“通常指的是一种网络应用程序的工作方式。

在采用 C/S 方式的网络应用中，运行在一个端系统上的客户进程总是主动向运行在另一个端系统上的服务器进程发出服务请求，服务器进程可接受来自多个客户进程的请求，并进行响应以提供服务。

采用 C/S 方式的网络应用有：万维网、电子邮件、文件传输 FTP 等。

人们习惯上把主要用来运行客户程序的计算机也称为 client，把主要用来运行服务器程序的计算机称为 server。server 通常是高性能计算机且 24h 开机，而 client 通常为普通计算机，它不一定总是处于开机状态。

# Peer to Peer

在 P2P 方式（对等方式）的网络应用中，通常没有固定的服务请求者和服务提供者，分布在网络中的应用进程是对等的，被称为对等方（有时将运行对等方软件的计算机也称为对等方）。

P2P 方式可支持大量对等用户（如上百万个）同时工作。BitTorrent 或 emule 都使用了 P2P 的工作方式。

# 因特网的核心部分

在网络核心部分起特殊作用的是 router，它是一种专用计算机（但不是主机）。router 是实现分组交换的（packet switching）的关键构件，其任务是转发收到的分组，这是网络核心部分最重要的功能。

# 电路交换

步骤（以打电话为例）：建立连接（分配通信资源）->通话（一直占用通信资源）->释放连接（归还通信资源）

TIP

当使用电路交换来传送计算机数据时，其线路的传输效率往往很低。因为计算机数据是突发式地出现在传输线路上的，因此线路上真正用来传输数据的时间往往不到 10%甚至 1%。实际上，已被用户占用的通信线路在绝大部分时间里都是空闲的。

# 分组交换

计算机网络通常采用分组交换技术。

  • 报文：要发送的整块数据。
  • 数据段：由较长的报文划分成（每个数据段是等长的）。
  • 首部：由必要的控制信息组成（如目的地址、源地址等）。
  • 分组（包）：一个数据段前面加上首部，就构成一个分组。
  • 包头：分组的首部。
  • 分组交换机：在因特网中，最重要的分组交换机就是 router，用来转发分组即进行分组交换。
# 存储转发

分组是因特网中传送的数据单元。计算机将分组通过通信链路直接发送给分组交换机，分组交换机接收到一个分组，先将分组暂时存储下来，再检查其首部，按照首部中的目的地址查找转发表，找到合适的端口（就是分组交换机和外部连接的接口）转发出去，把分组交给下一个分组交换机。这样一步一步经过多个分组交换机把分组转发到最终的目的计算机。

直白的讲，分组先传送到一个路由器，然后暂存一下，查找路由表，再转发到下一个路由器，这就是分组交换的“存储转发”的过程。

TIP

这里要注意的是，router 暂时存储的是一个个短分组，而不是整个的长报文。短分组暂存在 router 的存储器（即内存）中而不是存储在磁盘中，这就保证了较高的交换效率。

# 输出缓存（输出队列）

每个分组交换机有多条链路与之相连。对于每条相连的链路，该分组交换机有一个输出缓存，用于存储分组交换机准备发往哪条链路的分组。

# 排队

如果到达的分组需要从某条链路转发出去，但发现该链路正忙于传输其他分组，则分组必须在该输出缓存中等待。

# 分组丢失

当一个分组到达时发现输出缓存已满，到达的分组或已经排队的分组之一将被丢弃。

# 网络拥塞

当网络中有大量分组需要从某条链路转发时而出现的分组丢失的情况。

TIP

与电路交换不同，在分组交换在传输分组的过程中并非自始至终占用整个端到端的通信资源，而是逐段地占用，分组在哪段链路传输，就占用该链路资源。分组在传输时就这样一段接着一段 地断续占用通信资源，而且还省去了建立连接和释放连接的开销，因而数据的传输效率更高。

# 提高可靠性

为了提高分组交换网的可靠性，因特网的核心部分常采用网状拓扑结构，使得当发生网络拥塞或少数结点、链路出现故障时，router 可灵活地改变转发 route 而不致引起通讯的中断或全网的瘫痪。

# 分组交换的优点
优点 所采用的手段
高效 在分组传输的过程中动态分配传输带宽，对通信链路是逐段占用的
灵活 为每一个分组独立地选择转发 route
迅速 以分组作为传送单位，可以不先建立连接就能向其他主机发送分组
可靠 分布式多路由的分组交换网，使网络有很好的生存性
# 分组交换的缺点
  • router在转发分组时需要花费一定的时间，造成时延。
  • 无法确保通信时端到端所需的带宽，在通信量较大时可能造成网络拥塞。
  • 各分组必须携带的控制信息也造成了一定的开销。
  • 整个分组交换网需要专门的管理和控制机制。
